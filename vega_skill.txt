EXEMPLOS DE GRÁFICOS VEGA-LITE (sintaxe v5)

IMPORTANTE: SEMPRE estruture os dados dentro de "data": {"values": [...]}.
NUNCA coloque "values" na raiz do JSON. Sempre inclua "$schema".

---

1) GRÁFICO DE LINHA (evolução temporal, ex: faturamento por mês)

{
  "$schema": "https://vega.github.io/schema/vega-lite/v5.json",
  "title": "Evolução do Faturamento Mensal",
  "data": {
    "values": [
      {"mes": "2024-07", "faturamento": 21016.75},
      {"mes": "2024-08", "faturamento": 49998.18},
      {"mes": "2024-09", "faturamento": 52859.00}
    ]
  },
  "mark": {"type": "line", "point": true, "strokeWidth": 3},
  "encoding": {
    "x": {"field": "mes", "type": "ordinal", "title": "Mês"},
    "y": {"field": "faturamento", "type": "quantitative", "title": "Faturamento (R$)"},
    "tooltip": [
      {"field": "mes", "type": "ordinal", "title": "Mês"},
      {"field": "faturamento", "type": "quantitative", "title": "Faturamento (R$)", "format": ",.2f"}
    ]
  }
}

2) GRÁFICO DE BARRAS (comparação entre categorias, ex: vendas por produto)

REGRAS GERAIS:
- Use "$schema": "https://vega.github.io/schema/vega-lite/v5.json" sempre.
- Os dados devem SEMPRE estar em "data": {"values": [...]}.
- Use "tooltip" para melhorar a interatividade.
- Use "title" descritivo no gráfico e nos eixos.
- Para valores monetários, use format ",.2f" no tooltip.
- Escolha o tipo de gráfico de acordo com a pergunta:
  - Evolução no tempo -> line ou area
  - Comparação entre categorias -> bar
  - Comparação entre categorias e subcategorias -> bar agrupado (xOffset)
  - Proporção/participação -> arc (pizza/rosca)
  - Relação entre duas variáveis numéricas -> circle/point (dispersão)
  - Múltiplas séries no tempo -> line com "color"
